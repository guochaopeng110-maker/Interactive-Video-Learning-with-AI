## What to build

Build the first complete admin workspace slice spanning catalog management, user and role management, governance, and organization analytics.

This slice should establish the admin role as a governance workspace rather than an extension of teacher authoring. It should cover:

- organization landing or dashboard
- `Major` and `Course` catalog structure
- user and role assignment context
- publication governance and audit entry points
- organization-level analytics

The result should be a demoable admin path showing how organizational structure and governance relate to the same shared course model without exposing content-authoring responsibilities.

## Acceptance criteria

- [ ] An admin can move through catalog, users, governance, and analytics within one coherent workspace.
- [ ] The prototype makes the admin role visibly distinct from teacher authoring and student learning.
- [ ] Admin-facing screens focus on ownership, visibility, governance, and oversight rather than direct lesson editing.
