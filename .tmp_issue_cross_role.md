## What to build

Build the cross-role consistency and permission-boundary slice for the role-based prototype redesign.

This slice should validate and polish the places where teacher, student, and admin workspaces touch the same domain concepts. It should ensure that the prototype expresses one product model with role-specific action boundaries instead of three unrelated page collections.

The slice should cover:

- shared `Course` representation across roles
- shared `KnowledgeGraph` visual language with editable versus read-only differences
- shared `Lesson` framing across teacher preview and student playback
- permission-boundary checks for draft visibility, governance-only actions, and role-safe navigation

The result should be a demoable cross-role consistency pass that makes the redesigned prototype feel like one product.

## Acceptance criteria

- [ ] Shared domain concepts such as `Course`, `KnowledgeGraph`, and `Lesson` remain visually coherent across role workspaces.
- [ ] Teacher, student, and admin actions remain clearly separated by permission boundary in the prototype.
- [ ] The prototype navigation model reinforces role-safe visibility, especially around draft content and governance actions.
