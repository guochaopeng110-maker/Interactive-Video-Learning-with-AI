## What to build

Build the role-based prototype shell that replaces the old linear screen sequence with a stable route and navigation structure.

This slice should establish the shared entry layer and the role-aware prototype grouping needed by all later screens. It should give the prototype a complete path from login into role-specific workspaces while keeping shared product surfaces consistent.

The slice should cover:

- a unified login route for all roles
- an explicit role-entry or role-switching step after authentication
- route groups for teacher, student, and admin prototype workspaces
- shared workspace chrome such as top-level navigation, role context, and consistent global actions
- navigation contracts that later slices can plug into without redefining the structure

This is the structural tracer bullet for the role-based prototype redesign.

## Acceptance criteria

- [ ] A user can enter the prototype through a shared login route and reach a role-aware landing path.
- [ ] Teacher, student, and admin prototype routes are grouped under a consistent information architecture rather than the old linear sequence.
- [ ] Shared shell behavior and role context are visible and reusable for later prototype screens.
