## What to build

Build the first complete teacher workspace slice from `Course` entry through `Material` review into the teacher-facing `KnowledgeGraph`.

This slice should make the teacher authoring context coherent around one `Course`, then carry that context through:

- course overview and entry
- `Material` upload or review context
- AI suggestion review with explicit low-confidence focus
- transition into an editable teacher `KnowledgeGraph`

The result should be a demoable teacher path showing how confirmed `KnowledgePoint` emerge from `Material` and become usable inside the course structure.

## Acceptance criteria

- [ ] A teacher can move through a single `Course` context into `Material` review and then into the teacher `KnowledgeGraph`.
- [ ] The prototype visibly distinguishes high-confidence and low-confidence AI review states for teacher decision-making.
- [ ] The output of the slice is a teacher-facing `KnowledgeGraph` state that can logically feed later `Lesson` authoring.
