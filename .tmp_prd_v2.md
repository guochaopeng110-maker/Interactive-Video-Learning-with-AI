## Problem Statement

In nursing education, students often finish traditional video lessons with a weak transition from "I understand" to "I can do". Interaction is usually limited to static quizzes, leading to low engagement, low completion rates, and poor retention. Teachers also spend significant time preparing course materials, reducing time available for instructional design.

A second gap is navigability: students cannot quickly see which knowledge points exist, how they relate, or where to start based on interests and weaknesses.

## Solution

Build a school-oriented AI interactive course video platform for common nursing subjects.

Instead of producing one long linear video, the platform generates multiple knowledge-point assets (script draft, quiz draft, video-material draft) constrained by teacher-provided textbooks/handouts/curriculum standards. Teachers review and arrange these into branching interactive micro-lessons using a node-graph editor.

Students learn through a knowledge-graph guided entry point, choose relevant nodes, and enter interactive branching videos. During learning, an AI assistant answers questions with textbook-first grounded responses and explicit source awareness.

All publishable learning content is teacher-reviewed; unrestricted free-generation is out of scope.

## User Stories

1. As a nursing instructor, I want to upload textbooks and handouts, so that AI outputs stay aligned with approved teaching materials.
2. As a nursing instructor, I want AI to draft knowledge-point scripts, so that I can reduce first-pass content authoring time.
3. As a nursing instructor, I want AI to draft single-choice questions per node, so that I can quickly add checkpoint interactions.
4. As a nursing instructor, I want AI to draft scenario-based branch prompts (A/B), so that students can learn through contextual decisions.
5. As a nursing instructor, I want AI to generate draft video assets per knowledge point, so that I can assemble lessons faster.
6. As a nursing instructor, I want to review and edit all AI outputs before publish, so that content quality and safety remain under teacher control.
7. As a nursing instructor, I want a node-graph authoring experience, so that branch flow is easier to understand than timeline-only editing.
8. As a nursing instructor, I want to define remediation branches for wrong answers, so that students receive corrective explanations before returning to the main path.
9. As a nursing instructor, I want to publish a versioned interactive micro-lesson to a class, so that delivery is controlled and auditable.
10. As a nursing student, I want to see a knowledge graph of the course, so that I can understand concept structure and prerequisites quickly.
11. As a nursing student, I want to click a knowledge node and jump directly into its interactive lesson, so that I can learn by interest or weakness.
12. As a nursing student, I want node status indicators (not started/in progress/completed), so that I can track learning progress.
13. As a nursing student, I want to answer interaction questions during playback, so that I stay actively engaged.
14. As a nursing student, I want wrong-answer remediation immediately, so that I can fix misunderstandings before moving on.
15. As a nursing student, I want to ask the AI assistant while learning, so that I can resolve confusion without leaving the lesson.
16. As a nursing student, I want AI answers to prioritize current course materials, so that explanations match what my teacher expects.
17. As a nursing student, I want out-of-material explanations clearly labeled as extended reference, so that I can distinguish official from supplementary content.
18. As a course lead, I want completion-rate dashboards, so that I can evaluate instructional effectiveness.
19. As a course lead, I want interaction participation metrics per node, so that I can identify low-engagement sections.
20. As a course lead, I want pre/post assessment comparison on knowledge points, so that I can measure mastery lift.
21. As a school pilot owner, I want a small controlled rollout (1 school, 1 course, 2 teachers, 100 students, 4 weeks), so that we can validate outcomes before expansion.
22. As a school stakeholder, I want clear boundaries on AI generation policy, so that compliance and quality risks are reduced.
23. As a teacher, I want templates for branch structures, so that first-time setup has lower friction.
24. As a teacher, I want publishing to require completion of review checkpoints, so that accidental unreviewed release is prevented.
25. As a student, I want smooth continuity from knowledge map to lesson and assistant, so that learning flow is uninterrupted.

## Implementation Decisions

- Product scope is MVP-first for common nursing subjects in school settings, with primary learner users as nursing students and initial internal champion as department/course leaders.
- Core delivery unit is an interactive branching micro-lesson (2-3 branch nodes) rather than long-form linear videos.
- Content generation policy is "material-constrained generation + teacher review before publish" as mandatory baseline.
- Generation modes:
  - Default: constrained by uploaded textbooks/handouts/standards.
  - Allowed with label: limited extension explanations.
  - Disallowed: evidence-free unrestricted generation for publishable lesson content.
- Teacher authoring module is centered on a graph-based lesson composition model, not timeline-first editing.
- Branch learning logic for MVP: wrong answer routes learner to remediation explanation branch, then returns to mainline.
- Student learning entry is a lightweight course knowledge graph with node metadata and prerequisite hints.
- In-lesson assistant uses retrieval-grounded, textbook-first response strategy with source-aware output and explicit extension labeling when outside course corpus.
- Analytics module must capture at least three north-star metrics: completion rate, interaction participation rate, and mastery lift from pre/post checks.
- Rollout and validation envelope is fixed to a pilot boundary (1 school, 1 course, 2 teachers, 100 students, 4 weeks) for measurable delivery.
- Deep module opportunities to isolate and keep stable:
  - Content Grounding Engine: unifies material ingestion, chunking, retrieval, and citation mapping behind stable query interfaces.
  - Branch Progression Engine: encapsulates branch-state transitions, remediation routing, and resume logic as deterministic state behavior.
  - Learning Telemetry Engine: standardizes event collection and metric aggregation for course-level reporting.
  - Authoring Orchestration Engine: coordinates generate-review-compose-publish gates with policy checks.

## Testing Decisions

- Good tests focus on external behavior and policy guarantees, not internal implementation details.
- Priority test targets:
  - Content Grounding Engine: verifies textbook-first retrieval, source attribution presence, and extension-label behavior when corpus coverage is insufficient.
  - Branch Progression Engine: verifies deterministic transitions, wrong-answer remediation routing, and return-to-mainline behavior.
  - Authoring Orchestration Engine: verifies unreviewed content cannot be published and review gates are enforced.
  - Knowledge Graph Navigation behavior: verifies node state transitions and node-to-lesson jump behavior.
  - Learning Telemetry Engine: verifies event-to-metric aggregation for completion, interaction rate, and pre/post mastery lift.
- Test layering:
  - Unit tests for deterministic policy and state machines.
  - Contract/service tests for module boundaries and data contracts.
  - End-to-end pilot-flow tests for teacher publish and student learning loops.
- Prior-art note: because the repository currently contains little implementation-level test precedent, testing standards should be established alongside first module delivery and treated as reference patterns for subsequent issues.

## Out of Scope

- End-to-end fully automatic AI publishing without teacher review.
- Open-ended subjective scoring and advanced free-form grading workflows.
- Complex timeline-grade video editing capabilities.
- Cross-school content marketplace and distribution economics.
- Advanced adaptive recommendation and mastery-personalization beyond lightweight node guidance.

## Further Notes

- This PRD intentionally balances two buyer-facing values (teacher efficiency + student outcomes) while sequencing MVP validation around student learning metrics.
- The platform should use consistent domain language across future docs and implementation: knowledge point, branch node, remediation branch, material-constrained generation, and grounded assistant response.
- Architectural decisions with cross-cutting impact should be captured as ADRs as implementation begins.
