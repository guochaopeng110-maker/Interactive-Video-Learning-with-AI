# 半自动知识点提取

教师上传教材后，系统需要构建课程知识图谱。核心问题是知识点提取的自动化程度。

## 决策

采用 **半自动模式**：AI 从教材中提取候选知识点 → 教师在编辑器中确认、修改、增删 → 知识点入库并形成知识图谱。

## 拒绝的选项

- **全自动提取**：AI 一次性提取知识点，直接入库。虽然效率更高，但护理教材专业性强，AI 幻觉可能导致错误知识点进入系统，违反 PRD 确立的「teacher review before publish」原则。全自动模式下教师的信任感和可控感也会降低。

## 后果

- Content Grounding Engine 需要支持知识点提取 + 人工确认两个阶段。
- 前端需要一个知识点审核编辑器，展示 AI 建议的候选列表，支持批量确认和手动添加。
- Chunk 与 KnowledgePoint 的映射在教师确认后完成回填。
