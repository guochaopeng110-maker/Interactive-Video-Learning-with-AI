# Domain Documentation

This repo uses a **single-context** layout for domain knowledge.

## Locations
- **Core Context**: `CONTEXT.md` at the repo root.
- **Architectural Decisions**: `docs/adr/*.md`.

## Rules for Agents
1. **Read First**: Always check `CONTEXT.md` before proposing architectural changes.
2. **ADRs**: New significant architectural decisions should be recorded as a new ADR in `docs/adr/`.
