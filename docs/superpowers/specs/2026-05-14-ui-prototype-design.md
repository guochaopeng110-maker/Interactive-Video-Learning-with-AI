# UI 原型设计方案

## 目标

为护理教育 AI 互动视频学习平台产出完整 UI 原型，在 Issue #2 实现之前验证设计方向。

## 全局设计约束

- **视觉风格**: Modern Minimal（Notion/Linear 式，排版驱动，大面积留白）
- **配色方案**: 由 UI UX Pro Max 根据产品类型自动推荐
- **技术栈**: Next.js + Tailwind CSS（遵循 ADR 0001 TypeScript 全栈）
- **产出格式**: 9 个原型路由，每屏 3 个结构迥异的变体，通过 `?variant=` 切换

## 核心屏幕（9 屏，按用户流排序）

| # | 屏幕 | 角色 | 关键交互 |
|---|------|------|----------|
| 1 | 登录/注册 | 共享 | 邮箱+密码，注册→选角色 |
| 2 | 教师控制台 | 教师 | 课程列表，数据概览，快捷入口 |
| 3 | 教材上传+AI审核 | 教师 | 上传→分块→知识点提取→审核通过 |
| 4 | 知识图谱（教师视图） | 教师 | 选取知识点，构建课程范围 |
| 5 | 节点编排器 | 教师 | 拖拽编排 LessonNode，配置分支 |
| 6 | 课程预览 | 教师 | 学生视角预览发布前效果 |
| 7 | 学生选课 | 学生 | 浏览课程，加入学习 |
| 8 | 知识图谱（学生视图） | 学生 | 导航+进度+节点状态 |
| 9 | 互动播放器+AI助手 | 学生 | 视频+答题+分支+AI 助手侧边栏 |

## 执行节奏

逐屏深挖：Design System 补充 → 3 变体 → 用户反馈 → 融合 → 下一屏

## 前置步骤

运行 UI UX Pro Max 生成全局 MASTER.md 设计系统

## 判定标准

每屏的设计决策需明确记录（选择哪个变体方向及理由），为后续 Issue 实现提供 UI 约束。

## 已确认决策

| 屏幕 | 选择 | 理由 |
|---|---|---|
| 1 登录/注册 | B - Split Screen | 左侧品牌氛围 + 右侧高效表单，最适合首屏建立产品识别度 |
| 2 教师控制台 | A 侧边栏骨架 + B 横向课程卡片 | 兼顾后台骨架稳定感与课程内容浏览效率 |
| 3 教材上传+AI审核 | A - Linear Stepper Workspace | 最清晰地体现分步向导、AI 处理中状态和结果优先审核，且最能传达“流程不慢” |
